CastScope スコアとは

タレントの「社会的注目度・好感度・メディア適性」を6つの指標から数値化し、メディア種別に応じた重み付けで総合スコアを算出するキャスティング支援ツールです。データは毎週月曜 AM 6:00 に GitHub Actions で自動収集されます。

📊 6指標 📡 5データソース 📺 7メディア重み 🔄 毎週月曜 自動更新 👥 300名収録
📐6指標の詳細
①
Google Trends スコア
基本ウェイト 25%
何を測るか
過去90日間における検索関心度の高さ。Google Trends APIから取得し、0〜100で正規化。
なぜ重要か
検索需要はタレントへの「今の関心」を最もリアルタイムに反映する指標。出演発表・受賞・スキャンダルなどのイベント直後に急上昇する。
算出方法
collect_sns.py が Google Trends API を呼び出し、直近90日の週次データを取得 → 最大値で正規化（0〜100）→ SQLiteに保存。
活用シーン
テレビ 30%WEB 25%CM 25%
旬のタイミングでの起用判断に有効。
②
YouTube 平均再生数
基本ウェイト 25%
何を測るか
YouTubeの直近動画における平均再生数。YouTube Data API から公式チャンネルの最新20本を取得して平均算出。
なぜ重要か
動画コンテンツでの「実際に見られている数」を示す実績指標。フォロワー数より実態を正確に反映する。
算出方法
YouTube Data API → 最新20本の再生数を取得 → 平均値を算出 → log10 スケールで正規化（0〜100）。
活用シーン
SNS 30%映画 25%WEB 25%
動画コンテンツとの親和性判断に有効。
③
ニュース件数
基本ウェイト 15%
何を測るか
過去30日間にタレント名が登場したニュース記事の件数。Google News RSS から収集。
なぜ重要か
メディア露出の量的指標。件数が多いほど社会的認知が広がっている状態を示す。
算出方法
Google News RSS でタレント名を検索 → 30日間の件数をカウント → 件数÷3（上限100）として正規化。
活用シーン
新聞 35%映画 20%テレビ 20%
パブリシティ実績の判断に有効。
④
Wikipedia 閲覧数
基本ウェイト 10%
何を測るか
Wikipedia の該当タレントページの月間閲覧数。Wikipedia Pageviews API から取得。
なぜ重要か
「調べようとする人の数」＝認知の深さを示す指標。検索とは異なり、より能動的・詳細な関心を反映する。
算出方法
Wikipedia Pageviews API → 直近30日の合計閲覧数を取得 → 全タレント中の最大値で正規化（0〜100）。
活用シーン
映画 15%新聞 15%出版 10%
コアファン層の深さ判断に有効。
⑤
ニュース感情スコア
基本ウェイト 15%
何を測るか
ニュース記事の内容がポジティブかネガティブかを数値化したスコア。0〜100（100＝完全ポジティブ）。
なぜ重要か
件数が多くても炎上・批判記事が多ければブランドリスクになる。量（ニュース件数）と質（感情）を分けて評価することが重要。
算出方法
収集したニュース記事にポジ/ネガキーワードを適用 → ポジ率をスコア化（0〜100）→ 100に近いほどポジティブな報道が多い状態。
活用シーン
CM 30%出版 25%新聞 20%
ブランドセーフティの判断に最も直結する指標。
⑥
YouTube チャンネル登録者数
基本ウェイト 10%
何を測るか
YouTube 公式チャンネルの登録者数。固定ファン層の規模を示す「基盤指標」。
なぜ重要か
再生数が一時的に変動しても登録者数は安定した影響力を示す。長期起用・継続契約の判断に適している。
算出方法
YouTube Data API → チャンネル登録者数を取得 → log10 スケールで正規化（0〜100）。
活用シーン
SNS 25%出版 20%CM 10%
長期的なファン基盤の判断に有効。

🔢総合スコアの計算式
計算式
総合スコア = Σ（各指標の正規化値 × メディア別重み係数）
具体的な計算例（テレビ × ドラマ 選択時）
指標 実測値（正規化） 重み 計算
Google Trends70点30%70 × 0.30 = 21.00
YT平均再生数60点20%60 × 0.20 = 12.00
ニュース件数50点20%50 × 0.20 = 10.00
Wikipedia閲覧数80点10%80 × 0.10 =  8.00
ニュース感情75点15%75 × 0.15 = 11.25
YTチャンネル登録者40点5%40 × 0.05 =  2.00

🤖AI考察の読み方
📡 Layer 1｜メディア適性評価 常時表示
選択したメディア（テレビ/CM/映画など）において最重要な指標を中心に評価した考察。全タレント中の相対順位（上位X%）も表示。
例：CMでは感情スコアが30%ウェイトのため、「感情スコアが高く、ブランドへの好感転移が生まれやすい構成です」 などと評価。
🎯 Layer 2｜ジャンル適性評価 常時表示
メディアのサブ設定（ジャンル・ターゲット・形態）に応じた詳細考察。テレビ×ドラマとテレビ×バラエティでは重視ポイントが異なる。トレンド判定（高水準/安定/控えめ）も付加。
例：テレビ×ドラマ選択時 → 「感情移入度と話題継続性が鍵。Wiki PVが高く映画ファン層からの事前期待値が形成されている」 と評価。
⚠️ リスク評価 常時表示
感情スコアの平均値に基づく炎上リスクの3段階判定。感情スコアが低い個別タレントへのリスク警告も表示。
感情60点以上 → リスク低：ブランドセーフティは良好
感情40〜59点 → リスク中：継続モニタリングを推奨
感情40点未満 → リスク高：起用前に詳細調査を推奨
👥 主要ターゲット 常時表示
年代×性別の視聴意向マップデータから最高スコアのセグメントを特定して表示。F1層（20〜34歳女性）・M2層（40代男性）などの業界標準セグメントで表記。
例：「このキャスティングはF1層（20代女性）への訴求力が特に高い構成です（意向スコア73%）」
🤝 Layer 3｜キャスティング評価 2人以上で表示
複数タレントの組み合わせを評価。スコア差が小さい（20点未満）→ バランス型、大きい（20点以上）→ 対比型として判定。感情スコアが低いタレントにはリスク警告を表示。
例：「話題性（A）と安定感（B）の対比が明確で、異なる視聴者層を同時に取り込める戦略的キャスティングです」
🏆 総合推奨度 常時表示
相性スコア・バズスコア・メディア適性・リスクを総合した★1〜5の5段階評価。5が「即決推奨」、1が「再考推奨」。
★★★★★ → 即決推奨：数字・話題性・安全性の三拍子が揃っている
★★★☆☆ → 条件付き推奨：強みと改善点が明確。演出の工夫で評価が変わる
🔄データ更新スケジュール
⏰

毎週月曜日 AM 6:00 に GitHub Actions が自動実行され、300名分のデータを一括収集・更新します。Google Trends・YouTube・ニュース・Wikipedia の各 API を順次呼び出し、SQLite データベースに保存後、scores.json を再生成してGitHub Pages に自動デプロイします。

最新データの反映はダッシュボードのスコア更新日時でご確認ください。Google Trends APIのレート制限により、一部タレントのTrendsスコアが取得できない場合があります（その場合は前週の値を維持）。
